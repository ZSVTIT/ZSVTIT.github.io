服务注册与发现-Eureka

服务注册与发现的图示

调用关系说明

  1. 服务提供者在启动时，向注册中心注册自己提供的服务。
  2. 服务消费者在启动时，向注册中心注册自己的服务需求。
  3. 注册中心接收到服务消费者的服务需求后，返回符合其要求的服务提供者的地址给服务消费者
  4. 服务消费者访问服务提供者的地址去调用消费者

Eureka概述

  Eureka是一个服务注册和发现模块。

Eureka原理图

Eureka的组件

  Eureka采用C-S的设计架构。

  • Eureka Server:是提供服务注册的，各个节点启动后，会在Eureka Server中进行注册，这样Eureka Server的服务注册表中将会存储所有可用的服务节点的信息(主机与端口号、服务版本号、通讯协议)。某个服务不可用之后，其对应的注册信息就会从服务注册表中删除。

  • Eureka Client:是来处理服务的注册和发现的。当 Client 向 Server 注册时，它会提供一些元数据，例如主机和端口，URL，主页等。Eureka Server 从每个 Client 实例接收心跳消息（ping包）。 如果心跳超时，则通常将该实例从注册 Server 中删除。
