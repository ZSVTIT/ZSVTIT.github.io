服务提供者

概述

名词 概念
服务提供者 服务的被调用方，就是为其他服务提供服务的服务
服务消费者 服务的调用方，依赖于其他服务的服务

白话描述

  服务提供者其实就是直接操作数据库的，不直接对用户开放API，用户得通过服务消费者来调用服务提供者从而操作数据库。通常为了更好地操作数据库，都是要整合mybatis的。

创建服务提供者

  • 创建Maven项目管理文件pom.xml文件

  • 在src/main/java的某package下创建程序入口类ServiceAdminApplication.java

    @EnableEurekaClient表明该类是Eureka Client

  • 在src/main/resources目录下创建工程配置文件application.yml

      注意：需要指明 spring.application.name（应用程序名称），这个很重要，这在以后的服务与服务之间相互调用一般都是根据这个name的值(这里的名字可以自己取的，但是为了规范，就取该工程的名字)

      在这里我们直接用的程序名替代了具体的 URL 地址，在 Ribbon 中它会根据服务名来选择具体的服务实例，根据服务实例在请求的时候会用具体的 URL 替换掉服务名

  • 在src/main/java的controller的package下写Controller(就是具体提供什么服务)

    1.Format(String, Object, Object) 将指定的 String 中的格式项替换为两个指定的 Object 实例的值的文本等效项。

    2.@Value注解作用就是将我们配置文件的属性读出来

  • Eureka界面

      启动项目后，在浏览器输入地址localhost:8761
