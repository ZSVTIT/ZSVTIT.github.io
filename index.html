服务消费者(Ribbon)

概述

  服务消费者：服务的调用方。

  Ribbon 是一个负载均衡（ 分摊到多个操作单元上进行执行，这里的操作单元可以是服务器，这里的例子是端口号）客户端，可以很好的控制 http 和 tcp 的一些行为。

大白话

  服务消费者就是开放给用户来调用服务提供者的。

  Ribbon调用服务提供者的方法：在服务接口中

怎么调用服务提供者

  在服务类中(xxxService)中

  • 加上注入
  • 在方法中

Controller中的@RequestParm的解释

@RequestParam用于将请求参数区数据映射到功能处理方法的参数上。

RestTemplate

RestTemplate的介绍

是Spring用于同步client端的核心类，简化了与http服务的通信，并满足RestFul原则，程序代码可以给它提供URL，并提取结果。

Service中的getForObject()是RestTemplate请求方法中的一种。

RestTemplate的请求和Http请求的对比。

Http Method RestTemplate Method
DELETE delete
GET getForObject(返回消息体)
getForEntity
HEAD headForHeaders
OPTIONS optionsForAllow
POST postForLocation
postForObject
PUT put
any exchange
execute

如果只关注，返回的消息体，可以直接使用postForObject。

getForEntity方法的返回值是一个ResponseEntity<T>，ResponseEntity<T>是Spring对HTTP请求响应的封装，包括了几个重要的元素，如响应码、contentType、contentLength、响应消息体等。

@Autowired的用法和作用

这个注解可以对类成员变量、方法及构造函数进行自动装配的工作。通过`@Autowired`的使用来消除`set`,`get`方法。用了`@Autowired`的话，只需要声明即可。

服务提供者

发表于 2019-08-24 | 分类于 流水账

服务提供者

概述

名词 概念
服务提供者 服务的被调用方，就是为其他服务提供服务的服务
服务消费者 服务的调用方，依赖于其他服务的服务

白话描述

  服务提供者其实就是直接操作数据库的，不直接对用户开放API，用户得通过服务消费者来调用服务提供者从而操作数据库。通常为了更好地操作数据库，都是要整合mybatis的。

创建服务提供者

  • 创建Maven项目管理文件pom.xml文件

  • 在src/main/java的某package下创建程序入口类ServiceAdminApplication.java

    @EnableEurekaClient表明该类是Eureka Client

  • 在src/main/resources目录下创建工程配置文件application.yml

      注意：需要指明 spring.application.name（应用程序名称），这个很重要，这在以后的服务与服务之间相互调用一般都是根据这个name的值(这里的名字可以自己取的，但是为了规范，就取该工程的名字)

      在这里我们直接用的程序名替代了具体的 URL 地址，在 Ribbon 中它会根据服务名来选择具体的服务实例，根据服务实例在请求的时候会用具体的 URL 替换掉服务名

  • 在src/main/java的controller的package下写Controller(就是具体提供什么服务)

    1.Format(String, Object, Object) 将指定的 String 中的格式项替换为两个指定的 Object 实例的值的文本等效项。

    2.@Value注解作用就是将我们配置文件的属性读出来

  • Eureka界面

      启动项目后，在浏览器输入地址localhost:8761
